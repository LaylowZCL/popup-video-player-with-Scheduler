Manual de Utilização da Aplicação Desktop

1. Plataformas suportadas

  • Windows 10/11 (x64).
  • macOS (Intel/Apple Silicon).
  • Linux (x64, com ambiente gráfico compatível).

2. Modelo de autenticação (hardcoded)

Por decisão operacional, a aplicação desktop utiliza autenticação hardcoded para acesso à API.

  • Não há fluxo de login por utilizador final na app desktop.
  • As credenciais técnicas (API key e client ID) ficam embebidas no build.
  • Este modelo reduz suporte de campo em instalações de grande volume.

Nota: a rotação de credenciais exige novo build e redistribuição.

3. Regra obrigatória antes de compilar

A compilação só deve ocorrer depois de ser conhecido o domínio final da API.

  1. Definir o domínio final (ex.: https://dominiodaapi.com).
  2. Actualizar o valor hardcoded de BASE_URL para https://dominiodaapi.com/api.
  3. Validar endpoint de saúde/autenticação.
  4. Compilar os binários por sistema operativo.

4. Compilação e deploy por sistema operativo

O projecto possui três manifestos de empacotamento:

  • package.json - build principal para macOS.
  • WIN_package.json - build específico para Windows.
  • LNX_package.json - build específico para Linux.

Antes de compilar cada alvo, copiar o manifesto respectivo para package.json e executar o build.

Exemplo operacional:

  • Windows: usar configuração de WIN_package.json e gerar .exe/.msi.
  • Linux: usar configuração de LNX_package.json e gerar .AppImage/.deb.
  • macOS: manter package.json principal e gerar .dmg/.zip.

5. Instalação por sistema operativo

5.1 Windows

  1. Executar instalador assinado (.exe ou .msi).
  2. Concluir assistente e validar arranque automático, se aplicável.

5.2 macOS

  1. Abrir pacote .dmg e mover a app para Applications.
  2. Conceder permissões na primeira execução.

5.3 Linux

  1. Instalar pacote .AppImage, .deb ou equivalente.
  2. Garantir permissões de execução e codecs necessários.

6. Validação pós-instalação

  1. Confirmar autenticação com a API.
  2. Confirmar leitura de horários activos.
  3. Confirmar envio de eventos para /api/videos/report.

Nome oficial da aplicação: Banco Moc Popup Video.
